# 助听补偿的常用算法、术语与实现

这份文档面向“学过基础信号处理，但没系统做过助听器 DSP”的读者。

目标有三件事：

1. 解释 HearAdjust 当前 `+40 dB` 上限为什么来自实现约束，而不是医学阈值。
2. 解释助听器里常见的算法和术语到底是什么意思。
3. 解释这些算法在工程上通常是怎么实现的，以及和当前项目有什么差距。

## 1. 先把几个最容易混淆的概念分开

### 1.1 `dB HL` 不是 `dB gain`

- `dB HL` 是听力图里的“听阈损失”标尺。
- `dB gain` 是信号处理链里实际加到音频上的增益。

它们之间没有“一比一直接映射”。

原因包括：

- 不同频率的目标补偿不同。
- 软声、中声、响声的目标补偿不同。
- 助听器需要同时考虑可闻度、舒适度和最大输出限制。

所以“某频段损失 `40 dB HL`”并不等于“直接补 `+40 dB` 就是正确答案”。

### 1.2 EQ 不是助听器本体，而只是其中一个模块

`EQ` 是 equalization，中文一般叫均衡或均衡器。

从信号处理角度看，它本质上是在实现一个频率响应 $H(e^{j\omega})$，让不同频率分量经过系统后得到不同的增益。

最简单地说：

- 低频可以被放大
- 中频可以保持不变
- 高频可以被衰减

这就是 EQ。

但真正的助听器通常不只做静态 EQ，还会做：

- 动态压缩
- 输出限幅
- 反馈抑制
- 降噪
- 指向性处理
- 高频移频

因此，EQ 更像是助听器 DSP 里的一个基础模块，而不是全部。

### 1.3 静态补偿和动态补偿

静态补偿：

- 同一个频段，不管输入多大声，都给固定增益。

动态补偿：

- 输入很小声时，给较大增益。
- 输入已经很大声时，给较小增益。

真正助听器的核心，通常是后者。

## 2. HearAdjust 当前到底实现了什么

当前项目的助听模式，本质上是一个 8 频段串联 peaking EQ。

当前实现特征：

- 每个频段用一个 `BiquadFilterNode`。
- 滤波器类型是 `peaking`。
- 中心频率固定在 250, 500, 750, 1000, 2000, 4000, 6000, 8000 Hz。
- `Q = 1.5` 固定。
- 每个频段只改 `gain`，没有做动态检测。
- 整条链路近似为 `source -> 8-band EQ -> destination`。

这意味着它做的是：

- 一个参数均衡器
- 每个频段有固定中心频率、固定 Q、可变增益
- 所有频段串联，最终频响近似是各段频响相乘

它没有做：

- 分频后独立包络检测
- 宽动态范围压缩
- 输出级 limiter
- 自适应反馈抑制
- 高频频率搬移

因此，这一版更接近“个人补偿 EQ”而不是“完整助听器信号链”。

## 3. 什么叫 peaking EQ

`peaking EQ` 也叫 peaking filter、peak/notch equalizer、钟形 EQ。

它的频响形状通常像一个钟形：

- 当增益为正时，在中心频率附近形成一个“峰”
- 当增益为负时，在中心频率附近形成一个“谷”

它由三个主要参数决定：

### 3.1 中心频率 `f0`

决定“峰”或“谷”出现在哪个频率附近。

### 3.2 增益 `gain`

决定该频段是抬高还是压低。

- `+6 dB` 表示该频段幅度变大
- `-6 dB` 表示该频段幅度变小

### 3.3 品质因数 `Q`

决定影响范围有多宽。

- `Q` 越大，带宽越窄，改动越集中
- `Q` 越小，带宽越宽，改动越平缓

对于当前项目：

- `Q = 1.5` 是一个中等偏宽的值
- 这意味着每个频点影响的不只是一个精确频率，而是附近一整段频带

## 4. 为什么当前 `+40 dB` 是一个实现上限，而不是理论上限

当前实现只有静态 EQ，没有输出保护。这样做高增益时会带来几个问题。

### 4.1 多频段叠加会减少 headroom

如果多个频段都在提升，时域里的峰值可能明显增加。

哪怕每个频段单独看没有超过 0 dBFS，叠加后仍然可能在输出端造成：

- clipping
- inter-sample peak 风险
- 听感上的刺耳失真

### 4.2 噪声和伪影也会一起被放大

助听补偿不只会抬高有用语音，也会抬高：

- 背景噪声
- 压缩编码伪影
- 混响尾音
- 齿音和嘶声

### 4.3 静态 EQ 无法处理响度重振

感音神经性听损里常见的问题不是“整个动态范围都整体下移”，而是：

- 小声听不见
- 大声又可能很快变得不舒服

这就是为什么真正助听器通常要做 WDRC，而不是只做大固定增益。

### 4.4 没有限幅器时，系统稳定性和舒适度都不够好

在没有 limiter 或 MPO 控制的情况下，把 UI 上限继续放大到 `+50 dB` 或 `+60 dB`，工程上通常会让系统更脆弱，而不一定带来更好的主观收益。

## 5. 助听器常见术语和算法，按信号链来讲

下面按一个更典型的助听器 DSP 信号链来解释。

```text
输入
  -> 预处理 / 预加重 / 校准
  -> 分频器 / 滤波器组
  -> 每频段包络检测
  -> 每频段 WDRC
  -> 可选降噪 / 语音增强 / 方向性控制
  -> 可选高频移频
  -> 合成
  -> 输出限幅 / MPO
  -> 接收器
```

### 5.1 Fitting Formula，验配公式

常见的有：

- `NAL-NL2`
- `DSL v5`

它们不是“音频处理算法本身”，而是“目标生成算法”。

它们的输入通常包括：

- 听力图
- 年龄或用户类型
- 双耳情况
- 语言可懂度目标

它们的输出通常是：

- 某个频段在某个输入声级下应该给多少目标增益

所以，验配公式更像：

- 给 WDRC 提供目标曲线
- 给多频段压缩提供参数依据

而不是直接拿来做滤波器。

### 5.2 Filter Bank，滤波器组 / 分频器

这是多通道助听器的入口。

目的：

- 把宽带输入拆成多个子带
- 让每个频带可以独立处理

常见实现：

- IIR crossover
- FIR analysis filter bank
- FFT / STFT 子带法
- Gammatone 或 auditory-inspired filter bank

工程权衡：

- IIR：计算便宜，但相位和交叉点处理要小心
- FIR：线性相位更容易控制，但延迟和计算量更大
- STFT：做降噪和语音增强方便，但块处理会带来窗函数、重叠和延迟问题

### 5.3 Envelope Detector，包络检测器

WDRC 不是直接看原始波形，而是先估计每个频段的“能量”或“响度代表值”。

常见做法：

- 全波整流后低通
- RMS detector
- Hilbert envelope
- peak detector

设计关键点：

- 探测时间常数
- 峰值检测还是 RMS 检测
- attack / release 的平滑方式

峰值检测器：

- 对瞬态敏感
- 更适合保护峰值

RMS 检测器：

- 更接近平均能量
- 主观听感通常更平滑

### 5.4 WDRC，宽动态范围压缩

这是助听器里最核心的动态增益控制模块之一。

目标：

- 让软声变得可闻
- 让中等输入保持自然
- 让大声不至于过响

从输入输出曲线角度看：

- 低于某个 kneepoint，给较高增益
- 高于该点，随着输入增加，输出增长变慢

常见参数：

- `TK`，compression threshold 或 kneepoint
- `CR`，compression ratio
- `attack`
- `release`
- `make-up gain` 或 insertion gain

如果用分段线性方式描述，一种常见思路是：

- 当输入电平 `L_in` 小于 `TK` 时，使用线性增益 `G0`
- 当 `L_in` 大于 `TK` 时，增益按压缩比递减

压缩段的斜率常写成：

`1 - 1/CR`

当 `CR > 1` 时，输入每增加 `1 dB`，输出只增加小于 `1 dB`。

### 5.5 AGC，自动增益控制

很多文献里会把广义上的动态增益调节都叫 AGC。

但在助听器语境下，更常见的细分是：

- 慢速 AGC：更多负责整体响度适配
- 快速 WDRC：更多负责瞬时可闻度和舒适度控制
- 输出 AGC / limiter：负责峰值保护

所以 AGC 可以看成一个大类，而 WDRC 是其中非常典型的一类实现。

### 5.6 MPO，Maximum Power Output

MPO 关注的是“最大输出别太高”。

它的实现通常在链路后端。

作用：

- 限制大峰值
- 防止输出过载
- 保护主观舒适度

这通常不是为了改善语音清晰度，而是为了“保证系统不会太炸”。

### 5.7 Limiter 和 Compressor 的区别

二者本质都属于动态范围处理。

区别主要在于强度和目标：

- compressor：更平滑地改变动态范围
- limiter：以很高压缩比、很快响应限制峰值

工程上常见组合是：

- 每频段先做 WDRC
- 全带末端再做一个 fast limiter

### 5.8 Noise Reduction，降噪

助听器降噪通常不是为了“把噪声完全消掉”，而是为了：

- 提升舒适度
- 降低持续噪声负担
- 在不明显破坏语音的情况下改善主观体验

常见方法：

- spectral subtraction
- Wiener filtering
- MMSE 类估计
- modulation-domain filtering
- DNN 语音增强

助听器里通常要求：

- 延迟低
- 算法稳
- 不要产生强烈 musical noise

所以实际产品里往往偏保守。

### 5.9 Directionality / Beamforming，指向性 / 波束形成

如果设备有多个麦克风，就可以利用空间信息提升语噪比。

常见思路：

- 固定波束
- 自适应波束形成
- binaural beamforming

对语音理解很重要，因为很多时候“先提升输入 SNR”比“后面再加很多增益”更有效。

但浏览器标签页音频捕获场景通常没有真实声场麦克风阵列，因此这一块不适合直接照搬。

### 5.10 Feedback Cancellation，反馈抑制

助听器高增益时，一个大问题是声反馈：

- 扬声器发出的声音
- 又被麦克风重新收进去
- 形成闭环
- 最终出现啸叫

常见处理方法：

- 自适应滤波估计反馈路径
- 用 LMS / NLMS / 变体方法去跟踪反馈通道
- 从麦克风信号中减去估计出的反馈分量

工程难点：

- 真实反馈路径会随佩戴方式变化
- 与有用输入信号相关时容易误收敛

### 5.11 Frequency Lowering，高频移频 / 降频

当高频听损很重时，继续提高高频增益可能已经没有收益，甚至会带来：

- 噪声放大
- 反馈风险增加
- 齿音过强

这时会考虑 frequency lowering。

常见家族：

- frequency compression：把高频轴压缩到更低范围
- frequency transposition：把某些高频特征搬到更低区域
- frequency translation：更一般的频率映射

实现方法通常包括：

- 高频特征检测
- 频移或频率缩放
- 和原始频带做混合

难点在于：

- 很容易引入金属感
- 很容易破坏语音谱包络
- 参数需要对语种和用户很敏感

## 6. 从实现角度看，一个典型多通道 WDRC 怎么写

下面给一个更工程化的视角。

### 6.1 分带

先把输入拆成若干 band：

```text
x[n] -> x1[n], x2[n], ..., xM[n]
```

### 6.2 每带估计能量或包络

例如：

- 整流后低通
- 或在短窗上计算 RMS

得到每带电平估计 `L_i[n]`。

### 6.3 根据验配目标和压缩曲线算目标增益

对第 `i` 个频段，建立一个输入电平到目标增益的映射：

```text
G_target_i = f_i(L_i)
```

这个 `f_i` 通常来自：

- 听力图
- 验配公式
- 多个输入级别下的目标点拟合

### 6.4 用 attack / release 平滑增益

不能直接让增益瞬间跳变，否则会有：

- pumping
- zipper noise
- 明显调制失真

所以通常写成一个一阶平滑器，或者分别使用 attack 和 release 常数。

### 6.5 应用增益并重建输出

```text
y_i[n] = g_i[n] * x_i[n]
y[n] = sum_i y_i[n]
```

### 6.6 最后做输出保护

常见是：

- 全带 limiter
- output AGC
- MPO 控制

## 7. 常见实现选择及工程权衡

### 7.1 IIR vs FIR

IIR 优点：

- 计算量低
- 延迟低
- 很适合实时设备

IIR 缺点：

- 相位非线性
- 交叉频段设计要谨慎

FIR 优点：

- 可做线性相位
- 频响控制直观

FIR 缺点：

- 阶数高时延迟大
- 助听场景里延迟很敏感

### 7.2 时域滤波器组 vs STFT

时域滤波器组：

- 延迟容易控制
- 更适合传统低延迟助听器

STFT：

- 频率分辨率更灵活
- 适合做降噪和语音增强
- 但块处理通常会引入更多系统延迟

### 7.3 低延迟约束

助听器系统对延迟非常敏感。

延迟太高时，用户会感到：

- 回声感
- 自己说话发空
- 佩戴不适

这也是为什么很多助听算法实现时，会尽量避免过长窗长、过深缓冲和复杂迭代网络。

## 8. 回到 HearAdjust：当前项目和典型助听器差在哪里

当前 HearAdjust 的助听模式：

- 有固定频点 EQ
- 有手动增益控制
- 可以做基础补偿

但它还缺少典型助听器的几个核心能力：

### 8.1 缺少 level-dependent gain

当前同一个频段的增益与输入电平无关。

这意味着：

- 小声和大声被同样放大
- 无法有效模拟响度重振补偿

### 8.2 缺少输出级保护

当前没有独立 MPO 或 limiter。

所以高增益时，最直接的问题就是：

- clipping 风险
- 舒适度下降

### 8.3 缺少按耳、按场景、按输入级别的细粒度适配

真实助听器常见的是：

- 左右耳参数不同
- quiet / speech / noise 模式不同
- soft / medium / loud 输入目标不同

当前项目还没有这些维度。

## 9. 如果这个项目要往更像助听器的方向演进，合理路线是什么

### 阶段一：在现有 EQ 后加入输出保护

建议：

- 加 pre-gain headroom
- 加 post-EQ limiter 或 compressor
- 做峰值监控

这一步的意义：

- 先解决“高增益会炸”的工程问题

### 阶段二：引入多频段 WDRC

建议：

- 不再只改固定 gain
- 为每个频段引入包络检测和动态增益
- 增益曲线由听力图和验配公式决定

这一步的意义：

- 从“图形均衡器”走向“动态助听补偿器”

### 阶段三：加入更高级模块

例如：

- 高频移频
- 自适应降噪
- 左右耳独立参数
- 语境感知预设

## 10. 对当前项目最重要的结论

### 结论 1

当前 `+40 dB` 上限主要是实现约束，不是医学边界。

### 结论 2

如果没有 limiter、MPO 和 WDRC，单纯把滑块上限继续增大，往往只会让系统更容易失真、更吵、更不稳定。

### 结论 3

当补偿需求进一步提高时，下一步不应该是“继续加大静态 EQ”，而应该是“引入 level-dependent multiband processing”。

## 11. 术语速查表

- `EQ`：按频率改变增益的系统
- `peaking EQ`：围绕中心频率做钟形提升或衰减的滤波器
- `Q`：品质因数，控制带宽
- `filter bank`：把信号拆成多个频带的一组滤波器
- `WDRC`：宽动态范围压缩，对软声多放大、对响声少放大
- `AGC`：自动增益控制，广义动态增益调节
- `MPO`：最大输出控制
- `limiter`：强压缩、快响应的峰值限制器
- `beamforming`：多麦克风空间滤波
- `feedback cancellation`：反馈路径估计与抵消
- `frequency lowering`：把过高频率信息搬到更低可听区域
